# 模块: 实验检测报告 (experiment_report)

## 用途

从台账数据 + xlsx 模板生成实验检测报告。

## ⚠️ 核心原则

**模板里的数据是示例，不是真实数据。** 不要提取模板中的数值。

## 工具

### 纯 I/O（默认使用）

| 工具 | 说明 |
|------|------|
| `read_template_cells` | 读 xlsx 全部单元格位置+内容，不调 LLM |
| `fill_template_cells` | 写单元格值并另存，不调 LLM |

### LLM 辅助（仅在模板很复杂时使用）

| 工具 | 说明 |
|------|------|
| `analyze_experiment_template` | LLM 分析模板结构 |
| `confirm_template_analysis` | 确认模板分析中的不确定问题 |
| `generate_experiment_report` | LLM 从台账提取数据并填充 |

## 标准流程（你自己分析，不调 LLM）

```
1. read_template_cells(模板路径)
   → 返回:
     {
       "sheets": [
         {
           "name": "压实度试验检测报告",
           "rows": [
             {"row": 1, "cells": [{"cell": "A1", "value": "路基路面压实度..."}, ...]},
             ...
           ]
         },
         ...
       ]
     }

2. 你自己看:
   - 哪个是主报告 sheet？哪个是原始记录 sheet？
   - 固定文本有哪些？(试验室名称、检测依据的标准号...)
   - 哪些单元格需要填充数据？(桩号、压实度值、检测结论...)
   - 数据表在哪行？列头是什么？(序号/桩号/位置/压实度/结果判定)

3. 读台账:
   - 列: 序号/桩号/桩号及部位/检测项目/设计指标/厚度/施工日期/检测日期
   - 找到与报告对应的行(K9+320~K9+520 左幅水稳底基层试验段)

4. 你自己构建填充数据:
   {
     "压实度试验检测报告": {
       "C5": "K9+320.00～K9+520.00左幅水稳底基层试验段",
       "C4": "湖北工建基础设施建设有限公司",
       "L10": "20cm",
       "C7": "JTG 3450-2019",
       ...
     }
   }

   ⚠️ 固定文本(如标准号、仪器名称、判定依据)直接用模板中的值
   ⚠️ 台账有的列精确提取
   ⚠️ 台账没有的填"见原始记录"
   ⚠️ 检测结论: 你自己写，格式参考JTG F80/1-2017

5. fill_template_cells(模板路径, 输出路径, cell_values_json)
   → 生成最终报告
```

## 数据填充示例

假设模板第4行 C4 是"施工单位"，对应区域 id 为 `sheet1_region_4`:

```
台账: 有"施工单位"列但值是空的
→ 填模板中已有的示例值 "湖北工建基础设施建设有限公司"
→ 或填"见原始记录"
```

## 常见问题

**Q: 怎么判断哪些单元格是固定文本 vs 需要填充？**
A: 固定文本每个报告都一样（标准号、单位名称、签名行标签）。需要动态填充的是每份报告变化的（桩号、日期、检测值）。

**Q: 台账列和模板单元格不完全匹配怎么办？**
A: 从台账能直接对应的直接填。需要计算的计算后填。完全没有对应但报告必须有的，填"见原始记录"。

**Q: 模板有多个工作表怎么办？**
A: 主报告 sheet 是输出目标，其他 sheet 是参考（原始记录格式）。填充时只操作主报告 sheet。
